# AI创新药第一步检索 · 结果库

本库由 **Codex（执行检索）+ Claude（编排与质检）** 协作产生，依据
`AI创新药第一步检索Checklist_关键词与信息源.md` 组织。

## 目录结构

```
检索结果库/
├── 00_总览与进度/
│   ├── README.md                 ← 本文件
│   ├── 检索进度追踪.md            ← 19 个技术模块的执行 + 质检状态
│   ├── 检索记录_已检索关键词与网站.md ← 已搜过的查询式/网站（防重复，每模块追加）
│   └── 未能下载资料汇总.md        ← 全局「搜到但下不到」清单（最后集中处理）
├── 01_技术模块检索/
│   └── 方向X_.../X.Y_模块名/
│       ├── 检索结果.md            ← 六类检索结果（公司/技术/实验/资产/商业/负面）
│       └── 下载资料/              ← 论文/专利/公告 PDF、截图等原件
└── 02_最终输出/                   ← 第一阶段 8 份汇总交付物
```

## 每个模块 `检索结果.md` 必须包含的六类（中英文各做）

| 检索组 | 内容 |
|---|---|
| A 公司发现 | 2–4 家国外 + 1–3 家国内公司 |
| B 技术路线 | 模型/数据/实验/自动化 |
| C 实验验证 | 体外/体内/动物/湿实验 |
| D 资产验证 | Hit/Lead/PCC/IND/临床 |
| E 商业验证 | 合作/授权/首付款/里程碑（≥1 案例） |
| F 负面验证 | 失败/终止/裁员/转型（≥1 主动搜索） |

每条重大事实须记录：**发生日期 / 披露日期 / 来源链接 / 来源类型**
（公司自述 / 合作方确认 / 论文 / 专利 / 监管）。

## 质检（Claude 负责）

逐模块核对：六类是否齐全、中英文是否都做、信息源是否覆盖「必须检索」清单、
事实是否带日期与来源、下载资料是否归档、下不到的是否登记到全局汇总。
