# Codex 每模块检索提示词模板（v2，基于 1.1 试点修正）

## 试点教训
1. Codex 会在「搜索→下载→写文件」链条末尾耗尽预算，导致主文件 `检索结果.md` 未写出。
   → **改为：搜索完成后先写主文件，再做其他。下载交给 Claude。**
2. 无头 `codex exec` 下浏览器后端 `iab` 不可用、PowerShell 下载常报管道错、易超时。
   → **Codex 不下载，只在「待下载」表里给出可直接下载的直链；Claude 统一抓取。**
3. Stage 1 目标是覆盖率（宽进），用 14 字段「搜索发现表」作为标准结构。

## 调用命令（Claude 侧）
```
codex.exe exec --dangerously-bypass-approvals-and-sandbox \
  -c tools.web_search=true --skip-git-repo-check \
  -C "<检索结果库>" --json -o <last.md> - < <prompt.txt> > <log.jsonl> 2>&1
```
下载阶段由 Claude 读取各模块「待下载」表，用 curl.exe -L / WebFetch / archive.org / PMC 镜像统一抓取，存入该模块 `下载资料/`，并把真正抓不到的汇总到 `00_总览与进度/未能下载资料汇总.md`。
